---# TopVanAI 机器人配置指南

> 📖 本指南将手把手教你如何在钉钉和飞书中配置 TopVanAI 的 **IM 机器人**(双向对话)和 **Webhook 机器人**(单向推送),小白也能轻松上手!

**支持的平台:**
- ✅ 微信（WeChat）
- ✅ 钉钉 (DingTalk)
- ✅ 飞书 (Feishu/Lark)
- ✅ 企业微信（WeCom）
- ✅ QQ（QQ-BOT）

**配置类型:**
- 🤖 **IM 机器人**(Stream 模式): 双向对话,可以问答交互
- 📢 **Webhook 机器人**: 单向推送通知,适合定时推送、报警等场景

---

## IM 机器人 vs Webhook 机器人

### 核心区别

| 特性 | IM 机器人 (Stream 模式) | Webhook 机器人 |
|------|----------------------|---------------|
| **通信方向** | 双向对话(你 ↔ TopVanAI) | 单向推送(TopVanAI → 群聊) |
| **能否对话** | ✅ 可以问答、多轮对话 | ❌ 不能对话,只能推送通知 |
| **会话记录** | ✅ 有完整会话记录 | ❌ 无会话历史 |
| **配置难度** | ⚠️ 较复杂,需要企业认证和权限申请 | ✅ 超简单,1-2分钟搞定 |
| **适用场景** | 个人助手、问答交互 | 定时通知、报警推送、报表分发 |
| **多群使用** | 一个机器人可加多个群 | 每个群独立配置 |

**简单记忆:**
- **IM 机器人** = 你的私人助手(你问我答)
- **Webhook 机器人** = 你的广播电台(只播不收)

## 微信 IM 机器人配置

### 准备工作

在开始之前，请确保你已经:
1. 把有道龙虾升级到 2026.3.22 版本及以上
2. 微信的版本升级到 8.0.70（iOS 端）或 8.0.69（Android端）及以上

### 配置步骤

#### 步骤 1: 打开有道龙虾的微信的 IM 配置界面

打开有道龙虾，导航至：设置 -> IM 配置 -> 微信

![微信配置界面](./assets/001_86fd318d25.png)

#### 步骤 2: 打开微信连接二维码

在微信配置界面，点击“扫码连接微信”，即可弹出一个二维码

![扫码连接微信](./assets/002_706277d408.png)


#### 步骤3：使用微信进行扫码

使用微信扫描上述二维码，在弹出的页面中点击“连接”，之后等待龙虾段的页面刷新

![微信扫码后页面](./assets/003_f8ba0bbc77.png)


#### 步骤4：等待龙虾端页面刷新，并保存

![龙虾端页面刷新](./assets/004_41fca0cae4.png)


#### 步骤5：开始在微信指挥你的龙虾帮你干活

![微信指挥龙虾干活](./assets/005_498cd79abe.jpg)

✅ **必须选择 Stream 模式**

✅ **权限必须开通**

## 钉钉 Webhook 机器人配置

### 准备工作

在开始之前,请确保你已经:
- ✅ 已注册钉钉账号
- ✅ 已创建或加入一个群聊(可以是企业群或普通群)
- ⚠️ **必须使用电脑端或网页版进行配置** - 手机端无法创建自定义 Webhook 机器人

### 配置步骤

#### 步骤 1: 创建群聊(如果还没有)

在钉钉中创建一个新群聊,或使用现有群聊。

点击群聊右上角的设置按钮进入群设置:

![群设置入口](./assets/042_f46ade9b4b.png)

---

#### 步骤 2: 添加群机器人

在群设置中,找到 **群管理** → **机器人** → **添加机器人**。

选择 **自定义机器人(通过 Webhook 接入自定义服务)**:

![选择自定义机器人](./assets/043_85a28650c6.png)
![添加自定义机器人](./assets/044_b5cea74d1f.png)

---

#### 步骤 3: 配置机器人信息

填写机器人基本信息:

- **机器人名称**: TopVanAI (或任何你喜欢的名称)
- **机器人头像**: 可以上传龙虾 logo

---

#### 步骤 4: 配置安全设置

**重要:** 钉钉要求至少启用一种安全设置,推荐选择 **自定义关键词**:

**推荐配置:**
- ☑️ **自定义关键词**: 填写 `TopVanAI` 或 `AI`
  - 这样机器人发送的消息中必须包含这个关键词

**其他选项:**
- 加签 (Signature): 需要配置密钥,更安全但配置复杂
- IP 地址: 需要固定 IP,不适合个人用户

> **提示**: 如果选择"加签",需要记录下密钥 (Secret),后续配置需要用到。

---

#### 步骤 5: 获取 Webhook URL

创建完成后,会显示 Webhook 地址:

**Webhook URL 格式:**
```
https://oapi.dingtalk.com/robot/send?access_token=xxxxxxxxxxxxxx
```

**重要信息:**
- ✅ 复制完整的 Webhook URL
- ✅ 如果启用了"加签",同时复制 Secret 密钥
- ⚠️ 这些信息只显示一次,请妥善保管!

---

#### 步骤 6: 保存 Webhook 信息(推荐)

**💡 推荐做法**: 在 TopVanAI 客户端的 **设置 → 记忆** 中保存 Webhook 信息,方便后续直接使用。

**保存格式示例:**
```
我的钉钉TopVanAI群 Webhook: https://oapi.dingtalk.com/robot/send?access_token=你的access_token
```

这样以后你直接对 TopVanAI 说 **"给钉钉TopVanAI群推送消息"**,它就能自动从记忆中读取 Webhook 地址并执行!

---

#### 步骤 7: 测试机器人

**测试方法**: 直接把 Webhook 地址发给 TopVanAI,让它发一条测试消息。

**测试示例:**
```
https://oapi.dingtalk.com/robot/send?access_token=1babea08cc7d1a12a9ffb2711a2c18acdc15fb20000f9f3441dfba09426605e9

给我的钉钉机器人发个消息: Hello World
```

> **注意**:
> - 如果配置了关键词验证,推送的消息中必须包含关键词
> - Webhook 机器人是单向推送,群成员无法与机器人对话

---

### 钉钉 Webhook 配置要点

✅ **选择合适的安全验证方式**
- 推荐使用"自定义关键词",配置简单
- 加签验证更安全,但需要同时配置 access_token 和 secret

✅ **Webhook URL 格式要正确**
- 必须是完整的 URL,包含 `https://`
- 如果使用加签,需要单独填写 Secret 字段

✅ **注意群聊权限**
- 机器人只能在创建它的群中使用
- 如果需要多个群使用,需要在每个群中分别创建

## 常见问题与故障排查

### 钉钉 IM 机器人相关问题

**Q: 提示"Stream 连接失败"?**

A:
- 检查网络是否能访问钉钉 API
- 确认 Client ID 和 Client Secret 是否正确
- 重启 TopVanAI 应用重新建立连接

---

**Q: 机器人无法发送消息?**

A:
- 检查权限 `qyapi_robot_sendmsg` 是否已开通
- 确认应用已发布且版本为最新

---

### 飞书 IM 机器人相关问题

**Q: TopVanAI 客户端的"机器人"标签页是空白的,没有对话框?**

A:
这是最常见的问题,通常是由于配置步骤顺序错误导致的。

**解决方法:**
1. 确认配置顺序: 获取凭证 → 在 TopVanAI 中配置并建立连接 → 配置事件与回调 → 发布版本
2. 如果顺序错了,需要重新发布一个新版本(如从 1.0.0 升级到 1.0.1)
3. 重启 TopVanAI 客户端

---

**Q: 机器人收不到用户消息?**

A:
最常见的原因是没有订阅 `im.message.receive_v1` 事件。

**解决方法:**
1. 进入飞书开放平台的 **事件与回调** 页面
2. 在 **事件配置** 区域搜索 `im.message.receive_v1`
3. 点击 **添加事件** 按钮
4. 保存配置并重启 TopVanAI 客户端

> **提示**: 此事件是接收用户消息的核心权限,没有它机器人将无法接收任何消息!

> ⚠️ **重要**: 如果你是在应用已发布后才添加此事件,必须到 **版本管理与发布** 重新发布一个新版本(如从 1.0.0 升级到 1.0.1),配置才会生效!

---

**Q: 保存事件配置时提示"应用未建立长连接"?**

A:
说明 TopVanAI 客户端还没有启动或连接失败。

**正确做法:**
1. 先在 TopVanAI 中配置 App ID 和 App Secret
2. 重启 TopVanAI 客户端,等待连接建立
3. 再到飞书后台保存"事件与回调"配置

![飞书长连接状态](./assets/061_06708b6d9d.png)

---

### 钉钉 Webhook 机器人相关问题

**Q: 机器人推送失败?**

A:
1. 检查推送消息中是否包含配置的关键词
2. 确认 Webhook URL 是否正确复制(不要有空格或换行)
3. 检查 TopVanAI 是否在运行
4. 查看机器人是否被群管理员禁用

---

**Q: 提示"签名验证失败"?**

A:
- 检查 Secret 是否正确配置
- 确认 URL 和 Secret 分别填写在对应字段
- 注意不要把 `&secret=xxx` 拼接在 URL 中

---

**Q: 机器人提示"关键词不匹配"?**

A:
- 在推送消息中添加配置的关键词,例如: "TopVanAI: Hello World"
- 或者重新创建机器人,修改关键词设置

---

### 飞书 Webhook 机器人相关问题

**Q: 机器人推送失败?**

A:
1. 检查 Webhook URL 是否正确
2. 确认 TopVanAI 是否在运行
3. 查看群设置中机器人是否被禁用
4. 检查飞书是否有网络限制

---

**Q: 提示"webhook 地址不合法"?**

A:
- 检查 URL 是否完整,必须以 `https://open.feishu.cn/` 开头
- 确认没有多余的空格或特殊字符
- 重新复制 Webhook URL

---

### Webhook 机器人通用问题

**Q: 一个 Webhook URL 能在多个群使用吗?**

A:
- 不能。Webhook 机器人创建时就绑定了特定的群
- 如需多群使用,需要在每个群中分别创建机器人
- 然后在 TopVanAI 中配置多个 Webhook URL

---

**Q: 如何在 TopVanAI 中管理多个 Webhook?**

A:
**推荐方法**: 在 TopVanAI 的 **设置 → 记忆** 中保存所有 Webhook 信息。

**示例格式:**
```
技术TopVanAI群 Webhook: https://oapi.dingtalk.com/robot/send?access_token=xxx
运营TopVanAI群 Webhook: https://oapi.dingtalk.com/robot/send?access_token=yyy
产品飞书群 Webhook: https://open.feishu.cn/open-apis/bot/v2/hook/zzz
```

这样后续使用时,TopVanAI 能自动从记忆中读取对应群的 Webhook 地址!

---

**Q: Webhook URL 可以分享给别人吗?**

A:
- ⚠️ 不建议分享!
- 任何人获得 Webhook URL 都可以向群里发消息
- 如果泄露,应立即删除机器人并重新创建

---

```
3. Meta 开源最新多模态模型

```

### 最佳实践

1. **为每个群设置不同的关键词**
   - 技术群: `AI帮手`
   - 运营群: `小助理`
   - 这样可以更好地管理多个机器人

2. **合理设置推送频率**
   - 避免频繁推送,可能打扰群聊
   - 建议设置定时推送而非实时推送

3. **定期检查机器人状态**
   - 查看 TopVanAI 连接状态
   - 测试机器人推送是否正常

4. **备份 Webhook 信息**
   - 将 Webhook URL 保存在安全的地方
   - 记录每个 URL 对应的群聊名称
