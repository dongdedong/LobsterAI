## 一、主进程 (src/main/)

### 1.1 核心入口与生命周期

| 文件 | 大小 | 核心职责 |
|------|------|---------|
| `main.ts` | 388KB | **应用入口中心**：窗口管理、80+ IPC 注册、deep link 处理、系统托盘 |
| `preload.ts` | 50KB | **IPC 桥梁**：通过 `contextBridge` 暴露 `window.electron.*` 白名单 API |
| `sqliteStore.ts` | 27KB | **数据库层**：KV 存储 + 表迁移，底层 `better-sqlite3` 封装 |
| `coworkStore.ts` | 92KB | **会话 CRUD**：会话/消息的创建、更新、删除、分页查询 |
| `logger.ts` | 5KB | **日志系统**：拦截所有 `console.*`，写入 `electron-log` 滚动文件 |
| `windowState.ts` | 6KB | **窗口状态**：尺寸/位置的持久化与恢复 |
| `trayManager.ts` | 5KB | **系统托盘**：托盘图标菜单管理 |
| `autoLaunchManager.ts` | 1.5KB | **开机自启动**：注册/注销自启动项 |

### 1.2 Agent 引擎层 (src/main/libs/agentEngine/)

| 文件 | 大小 | 核心职责 |
|------|------|---------|
| `coworkEngineRouter.ts` | 8KB | **引擎路由器**：实现 `CoworkRuntime` 接口，将所有会话操作路由到 OpenClaw |
| `openclawRuntimeAdapter.ts` | **345KB** | **OpenClaw 适配器**：与 Gateway WebSocket 通信，转换事件格式 |
| `types.ts` | 5KB | **类型定义**：`CoworkRuntime` 接口、`PermissionRequest`、`CoworkContextUsage` |
| `coworkContinuityCapsule.ts` | 17KB | **上下文延续**：会话延续时的上下文封装与传递 |
| `coworkTopKEvidence.ts` | 10KB | **证据检索**：从会话历史中提取 Top-K 相关证据 |
| `subagentTracker.ts` | 32KB | **子 Agent 追踪**：追踪父会话下产生的子 Agent 执行历史 |

### 1.3 OpenClaw 引擎管理 (src/main/libs/)

| 文件 | 大小 | 核心职责 |
|------|------|---------|
| `openclawEngineManager.ts` | 62KB | **引擎生命周期**：安装检测、Gateway 启动/重启/停止、健康检查、端口扫描 |
| `openclawConfigSync.ts` | **131KB** | **配置同步**：将 GUI 设置同步到 OpenClaw 的 `openclaw.json` 配置文件 |

| `openclawChannelSessionSync.ts` | 27KB | **通道同步**：同步 OpenClaw Channel 会话到本地 SQLite |
| `openclawTokenProxy.ts` | 6KB | **Token 代理**：为 Gateway 提供 LLM API Token |
| `openclawAgentModels.ts` | 9KB | **模型注册**：将 GUI 配置的多 Provider 模型注入 OpenClaw |
| `coworkOpenAICompatProxy.ts` | 92KB | **OpenAI 兼容代理**：提供 OpenAI API 兼容的本地端点供 Gateway 调用 |
