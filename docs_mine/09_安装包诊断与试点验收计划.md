# 安装包、诊断与客户试点验收计划

> 生成日期: 2026-06-21  
> 阶段定位: 无后端 MVP 的第 5 阶段  
> 目标: 把“开发环境可运行”推进为“可发给种子客户安装试用”的交付包。

## 1. 阶段目标

阶段 5 不再以新增大功能为主，而是验证产品能否离开开发机:

1. Windows 安装包可以构建、安装、启动和卸载。
2. 干净 Windows 环境不需要源码、不需要开发依赖、不需要 `venusAI_Server`。
3. 客户可以导入 License、配置模型 Provider，并跑通一次 Cowork。
4. 出问题时，研发能通过日志和状态信息定位到模型、授权、网络、OpenClaw 或客户端打包问题。

## 2. 打包前自动预检

每次准备打包前先运行:

```powershell
npm run mvp:preflight
```

预检会检查:

| 检查项 | 目的 |
| --- | --- |
| `package.json` 作者信息 | 防止外发包残留上游联系邮箱 |
| Local BYOK 默认端点 | 防止默认请求上游服务或 `127.0.0.1:8787` |
| 服务端模型模式 | 防止默认注入服务端模型代理 |
| Youdao Provider 可见性 | 防止 MVP 模型列表暴露上游 Provider |
| NIM / NetEase Bee / POPO 可见性 | 防止 MVP IM 设置页暴露遗留网易系连接器 |
| `youdaonote` 默认状态 | 防止默认引导客户访问有道云笔记服务 |
| License 私钥 | 防止签名私钥进入客户端 |
| 第三方声明 | 确保商业化复核清单存在 |
| Git Bash | 确保 Windows 打包时 OpenClaw runtime 使用 Git Bash，而不是 WSL bash |

预检有 warning 不一定阻断打包，但必须记录到阶段汇报。当前已知 warning 是 `package.json` 的 OpenClaw 插件元数据仍包含网易系遗留插件，这些能力已经在 MVP 可见入口隐藏，商业化前仍需单独授权复核。

Windows 打包必须使用 Git Bash / MSYS bash。`C:\Windows\System32\bash.exe` 和 `WindowsApps\bash.exe` 是 WSL bash，不能用于当前 OpenClaw runtime 打包链路。

## 3. Windows 打包命令

打包前建议执行:

```powershell
npm run mvp:preflight
npm run build
npm run compile:electron
```

确认基础构建通过后，再运行:

```powershell
npm run dist:win
```

`dist:win` 会执行:

1. `predist:win`: 构建并同步 Windows x64 OpenClaw runtime。
2. `setup:python-runtime`: 准备 Windows 内置 Python runtime。
3. `build`: TypeScript + Vite 前端构建。
4. `compile:electron`: Electron 主进程编译。
5. `build:skills`: 构建内置 skills。
6. `electron-builder --win --x64`: 生成 Windows 安装包。

当前 MVP 内部试点包默认关闭 Windows 证书自动探测，并跳过 Windows 可执行文件 sign/edit 步骤，生成未签名安装包。这样可以避免无签名证书的开发机在解压 `winCodeSign` 时因为符号链接权限失败。正式对外发布前需要补充代码签名证书、发布主体、文件元数据校验和下载页信任说明。

## 4. 干净环境验收清单

建议用一台无源码、无 Node 开发环境、未启动 `venusAI_Server` 的 Windows 机器验证。

| 步骤 | 验收标准 | 证据 |
| --- | --- | --- |
| 安装 | 安装包可正常安装，无系统拦截或缺依赖错误 | 安装截图、安装耗时 |
| 启动 | 首次启动可进入主界面，无阻塞弹窗 | 主界面截图 |
| License | 设置页可看到机器码，可导入试用或正式 License | License 页截图 |
| BYOK 配置 | 可配置 DeepSeek / OpenAI-compatible / Ollama / LM Studio 之一 | 模型配置截图 |
| 连接测试 | 正确 Key 成功，错误 Key 给出可理解提示 | 成功/失败截图 |
| Cowork | 可创建会话并完成一个真实任务 | 会话结果截图 |
| 日志导出 | 设置页可导出 zip 日志包 | 日志 zip 文件名 |
| 卸载 | 程序可卸载，不影响用户手动保留的数据 | 卸载结果 |

## 5. 试点任务脚本

种子客户首次试用建议只验证 3 个高价值任务:

1. 基于一个本地 Excel 或 CSV 文件生成分析结论。
2. 基于一个本地文档生成汇报提纲或 PPT 草稿。
3. 在一个本地项目目录里完成只读代码分析或小范围修改。

不要在第一轮试点承诺以下能力:

1. 账号体系、团队管理、在线充值。
2. 公司统一模型代理。
3. 在线 Skill / Kit / MCP 市场。
4. 实时 ASR、媒体生成、HTML 公开分享。
5. Computer Use runtime 内置能力。

## 6. 问题归因表

| 现象 | 优先判断 | 需要收集 |
| --- | --- | --- |
| 应用打不开 | 安装包、系统权限、Electron 主进程 | 主日志、Windows 事件、安装截图 |
| Cowork 启动失败 | License、OpenClaw runtime、模型配置 | License 状态、OpenClaw 状态、cowork.log |
| 模型连接失败 | API Key、base URL、模型名、网络代理 | 连接测试结果、模型配置截图 |
| 工具调用失败 | 权限审批、文件路径、OpenClaw 执行环境 | 会话日志、权限弹窗截图 |
| 技能执行失败 | skill 依赖、Python/Node runtime、网络 | skill 日志、导出日志 zip |

## 7. 给老板的阶段汇报口径

本阶段交付物不是“又加了几个功能”，而是证明 MVP 可以真实交付:

1. 一个可安装的 Windows 包。
2. 一套打包前自动预检。
3. 一份干净环境验收记录。
4. 一份客户安装和配置文档。
5. 一份试点问题清单和付费场景反馈。

如果这五项都通过，就可以进入 1-3 个种子客户试点；如果安装或首次 Cowork 失败率高，就先修交付链路，不继续扩展云功能。
