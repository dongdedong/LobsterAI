# 方案 A 客户端提交变更说明

> 日期: 2026-06-20
> 范围: `E:\code\TopVanAI` 当前工作树中与方案 A、自建轻量服务、本地/BYOK 默认模式、日志优化、TopVan 品牌归属相关的代码和文档改动。
> 目的: 作为提交前审阅、PR 描述和后续验收的依据。

## 1. 总体结论

本轮改动把客户端从“默认依赖有道/网易云服务”调整为“默认本地 / BYOK 模式”，并补充了自建轻量服务的配置入口。

核心变化:

1. 新增统一的远程服务配置模型，集中管理官方、自建、本地/BYOK 三种服务模式。
2. 默认配置改为 `local-byok`，服务端模型默认禁用，避免新安装状态自动回落到有道服务端模型。
3. 主进程和渲染进程的更新、文档、商城、Portal、服务协议、HTML 分享等 URL 改为从统一配置解析。
4. 本地/BYOK 模式下禁用或短路依赖云服务的能力，包括服务端模型、媒体生成、实时 ASR、HTML 分享和启动阶段服务端缓存预热。
5. OpenClaw 配置同步不再在服务端模型禁用时注入 `lobsterai-server` provider。
6. 将用户可见的网易有道版权/出品/开发归属文案替换为 TopVan。
7. 补充测试，覆盖默认端点不指向有道/网易、端点解析、自建模式、日志导出可选缺失项、OpenClaw provider 注入边界等关键行为。

## 3. 远程服务配置设计

新增 `RemoteServicesMode`:

| 模式 | 值 | 行为 |
| --- | --- | --- |
| 官方服务 | `official` | 仅在用户显式选择时使用有道官方/测试端点。 |
| 自建服务 | `self-hosted` | 使用用户填写的自建服务地址，自动推导 `/catalog/*.json`、`/docs`、`/download-list`、`/service-terms` 等路径。 |
| 本地/BYOK | `local-byok` | 默认模式，使用 `http://127.0.0.1:8787`，禁用服务端模型和云能力。 |

新增 `ServerModelMode`:

| 模式 | 值 | 行为 |
| --- | --- | --- |
| 官方模型 | `official` | 用于显式官方服务模式。 |
| 自建代理模型 | `managed-proxy` | 用于自建服务代理模型能力。 |
| 禁用服务端模型 | `disabled` | 本地/BYOK 默认行为，只使用用户配置的第三方或本地模型。 |

默认解析规则:

1. 缺少 `app.remoteServices` 时默认走 `local-byok`。
2. `local-byok` 默认服务地址为 `http://127.0.0.1:8787`。
3. `self-hosted` 会清理尾部 `/`，并从 `serverApiBaseUrl` 推导文档、商城、更新和分享路径。
4. `LOBSTERAI_SERVER_BASE_URL` 可覆盖服务端基础地址，便于测试或部署环境注入。
5. 只有显式 `mode: official` 时才使用有道官方/测试端点。

## 4. 客户端行为变化

### 4.1 设置页

设置页的“通用”页签新增“远程服务”配置卡片:

1. 服务模式: 官方服务 / 本地 BYOK / 自建服务。
2. 自建服务地址: 非官方模式下可填写，保存前校验 HTTP/HTTPS URL。
3. 服务端模型: 官方模型 / 自建代理模型 / 禁用服务端模型。
4. 本地/BYOK 模式显示提示，说明服务端模型、媒体生成、实时语音和 HTML 分享会被禁用。

保存配置时会把 `remoteServices` 写入 `app_config.app.remoteServices`。

### 4.2 关于页和协议入口

关于页中的用户手册、官方社群、服务条款不再硬编码有道 URL，而是来自统一端点解析:

| 入口 | 本地/BYOK 默认 |
| --- | --- |
| 用户手册 | `http://127.0.0.1:8787/docs/lobsterai_user_manual` |
| 社群/关于 | `http://127.0.0.1:8787/about` |
| 服务条款 | `http://127.0.0.1:8787/service-terms` |

隐私/服务协议弹窗同样使用 `getServiceTermsUrl()`。

### 4.3 登录和额度

本地/BYOK 模式下:

1. 不打开官方 Portal 登录。
2. 不拉取 `/api/user/profile`、`/api/user/quota`、`/api/models/available`。
3. 前端使用本地虚拟用户 `Local BYOK User` 和本地额度状态。
4. 服务端模型列表被清空，避免 UI 或 OpenClaw 隐式选择 `lobsterai-server`。
5. 配置切换时监听 `config-updated`，重新应用本地或远程登录状态。

### 4.4 服务端模型和 OpenClaw

关键目标是阻止服务端模型禁用时的隐式回退:

1. `claudeSettings.ts` 中 `tryLobsteraiServerFallback()` 增加 `isServerModelModeEnabled()` 判断。
2. OpenClaw config sync 在 `serverModels.length === 0` 时不再注入 `lobsterai-server` provider。
3. 启动缓存预热仅在有 token 且服务端模型启用时执行。
4. 媒体生成插件在云能力不可用时写入 `enabled: false`，不再保留可调用配置。

### 4.5 云能力门控

本地/BYOK 模式下以下能力会被短路或禁用:

| 能力 | 行为 |
| --- | --- |
| 服务端定价目录 | 返回空模型列表。 |
| 服务端模型列表 | 返回空模型列表。 |
| HTML 分享 | 返回 `LOCAL_BYOK_CLOUD_FEATURE_DISABLED`。 |
| 媒体生成模型 | 返回空模型列表。 |
| 媒体任务状态/取消 | 返回禁用结果。 |
| Agent 媒体工具 | 返回 tool error，提示本地/BYOK 禁用媒体生成。 |
| 实时 ASR | 返回 `AsrApiCode.Disabled`。 |
| 媒体生成 entitlement | 切到本地/BYOK 后清空缓存。 |

### 4.6 本地资源替换

以下默认资源切到 `127.0.0.1:8787`:

1. 自动更新和手动更新 catalog。
2. Skill Store、Kit Store、MCP Marketplace catalog。
3. Computer Use runtime 下载包、Kit 下载包、Kit 图标。
4. IM 配置指南文档链接。
5. NIM QR 兼容入口。

Skill/Kit/MCP marketplace 拉取改用 `fetch`，可以同时支持 HTTP 本地服务和 HTTPS 官方服务。

### 4.7 日志导出优化

日志导出新增可选缺失项:

1. `install-timing.log` 标记为 optional。
2. 只缺 `install-timing.log` 时不再显示“关键日志缺失”警告。
3. 返回值增加 `optionalMissingEntries`，UI 仍根据 `missingEntries` 判断是否展示部分成功提示。

### 4.8 Python 运行时日志优化

开发环境缺少打包内置 Python 时改为 warn，避免 dev 日志里出现误导性的 error。正式打包环境仍然使用 error，保证真实资源缺失可见。

### 4.9 TopVan 品牌归属

已替换的用户可见归属:

1. 关于页版权主体。
2. 服务协议标题和描述。
3. Artifact 预览打开方式中的“有道龙虾浏览器”。
4. Cowork 分享图页脚。
5. OpenClaw 默认身份文件。
6. README、README_zh、LICENSE、docs_mine 中的出品/维护/Copyright 主体。

保留项:

1. 有道/网易服务端依赖分析文档中的历史技术描述。
2. 官方模式和测试模式配置中的官方端点。
3. 测试样本中用于验证“只有显式 official 才使用官方端点”的 URL。
4. 网易 IM / 网易云信 / NetEase Bee 等第三方集成名称。

## 6. 风险和注意事项

1. 默认模式变为本地/BYOK 后，新安装客户端会依赖 `http://127.0.0.1:8787` 提供文档、更新 catalog、Skill/Kit/MCP catalog 等轻量静态接口。
2. 如果本地轻量服务未启动，客户端核心聊天/BYOK 模型仍可配置，但文档、商城、更新检查等入口会请求失败。
3. 官方模式仍保留有道/网易端点，便于回归测试和兼容旧能力；这不是默认路径。
4. 媒体生成、实时 ASR、HTML 分享在本地/BYOK 模式被明确禁用；如果自建服务后续实现这些能力，需要扩展 `areCloudRemoteFeaturesEnabled()` 或增加更细粒度能力开关。
5. Computer Use runtime 和 Kit 下载已切到本地 URL，需要自建服务实际提供对应文件和正确 SHA256 校验对象。
6. NIM QR 默认入口切到本地兼容地址，如果仍要保留网易云信直连，需要通过配置或后续能力开关恢复。
7. 本轮没有删除官方端点常量，只是把默认路径切到本地/BYOK；仓库搜索仍会在官方配置、测试和依赖分析文档中看到有道/网易字样。
8. 当前工作树里包含方案 A 规划文档、客户端代码、品牌清理和日志优化，提交时可以选择一次提交，也可以按功能拆成多次提交。
